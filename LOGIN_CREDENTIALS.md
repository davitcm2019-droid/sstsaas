# Credenciais de Login (Exemplo)

Por segurança, este repositório não publica credenciais reais (email/senha) para acesso.

## Como testar localmente

1. Inicie o backend e o frontend.
2. Crie um usuário na tela de registro (`/register`).
3. Faça login na tela de login (`/login`).

## Observações

- Os dados são mock (em memória) e são perdidos ao reiniciar o servidor.
- Para cenários corporativos (admin/roles), a criação de usuários privilegiados deve ser feita por fluxo administrativo e/ou via banco de dados (fase de persistência).
